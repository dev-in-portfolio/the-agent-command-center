# MVP-30 — Product Narrative

The product starts with authenticated request and feedback review. Those signals are synthesized into product decisions, converted into roadmap and request drafts, and then presented through a guided demo control room.

This is a human-operated workflow:
- review the feedback,
- synthesize the signal,
- map the capability story,
- package the release narrative,
- and export a safe pitchable bundle.

What is intentionally blocked:
- service-role exposure,
- browser persistence,
- direct browser Supabase access,
- update/delete/approve/execute,
- automation,
- deploy/merge/push/PR controls.
