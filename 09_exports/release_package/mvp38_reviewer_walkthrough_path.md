# MVP-38 Reviewer Walkthrough Path

## Path Overview

- **Walkthrough ID:** mvp38-walkthrough-final
- **Reviewer Pool:** founder, recruiter, technical reviewer, operator
- **Path Type:** sequential audience-gated
- **Estimated Total Time:** 12 minutes (all audiences)
- **State:** locked for review execution

## Step-by-Step Walkthrough

### Step 1: Entry Gate
- Load the review room from the release manifest
- Verify all 7 export files are present and correctly named
- Confirm JSON manifest checksums match file list
- **Gate:** proceed only if all files are present and valid

### Step 2: Demo Script Review
- Open mvp38_demo_script_lock.md
- Walk through each click-by-click instruction
- Verify the walkthrough path matches the demo flow
- Check that timing estimates are annotated
- **Gate:** all instructions must be unambiguous and executable

### Step 3: Audience Path Review
- Open mvp38_final_review_audience_paths.md
- Review each of the four audience paths independently
- Verify founder path tells the strategic story
- Verify recruiter path reflects team and role context
- Verify technical reviewer path covers system interactions
- Verify operator path covers the full gate sequence
- **Gate:** each path must be independently coherent

### Step 4: Checklist Review
- Open mvp38_release_go_no_go_checklist.md
- Verify all checklist items are actionable and scoped
- Confirm audience sign-off rows are structured for completion
- **Gate:** no placeholder or incomplete checklist items

### Step 5: Timing and Talking Points Review
- Open mvp38_demo_timing_talking_points.md
- Validate timing estimates against click-by-click instructions
- Confirm talking points are mapped to each demo section
- **Gate:** timing must sum to within 10% of total estimate

### Step 6: Copy Bank Review
- Open mvp38_final_release_review_copy_bank.md
- Spot-check copy for consistency across all files
- Verify no contradictory instructions or stale references
- **Gate:** copy must be internally consistent and free of drift

### Step 7: Operator Gate
- Review all sign-off marks across checklist
- Make go/no-go determination
- Record decision and rationale in checklist summary
- **Gate:** this is the final and binding review step

## Reviewer Notes

- Each reviewer should annotate their findings directly in their path section
- If a gate fails, do not proceed to the next step — escalate to operator
- After operator sign-off, no further changes are permitted without a full restart
- All reviewer notes become part of the release record
- Estimated time per reviewer path section: 2-3 minutes
