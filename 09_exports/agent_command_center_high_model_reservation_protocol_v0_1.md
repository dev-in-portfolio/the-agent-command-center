# Agent Command Center High-Model Reservation Protocol v0.1

## Current Context
- Station Chief runtime is parked at v4.7.0.
- This protocol defines which work should be reserved for top-tier models.
- This document does not resume Station Chief.
- This document does not create v4.8.
- This document does not modify runtime behavior.

## Purpose
This document preserves complex architecture, validator, runtime, and execution-layer tasks for higher-capability sessions.

- this is a reservation protocol only
- it does not select which task happens next
- it does not recommend timing
- it does not authorize runtime behavior
- it does not grant permissions
- it does not create v4.8

## High-Model Principle
- high-model work is used for deep architecture
- high-model work is used for runtime changes
- high-model work is used for validator reasoning
- high-model work is used for safety-sensitive transitions
- high-model work still requires explicit operator direction
- high-model capability does not override operator authority

## Reserved Work Categories

- **Station Chief v4.8+**
  - why reserved: Requires deep runtime logic.
  - risk if handled by weak model: Logic bugs, parking violation.
  - approval required: Explicit runtime build token.
  - current status: Reserved/Parked
  - runtime effect if authorized later: Runtime modification.

- **runtime architecture**
  - why reserved: Affects overall system stability.
  - risk if handled by weak model: Unpredictable behavior.
  - approval required: Explicit architecture design token.
  - current status: Reserved
  - runtime effect if authorized later: Significant structural change.

- **validator redesign**
  - why reserved: Critical safety boundary enforcement.
  - risk if handled by weak model: Safety boundary bypass.
  - approval required: Explicit validator redesign token.
  - current status: Reserved
  - runtime effect if authorized later: Safety logic change.

- **worker routing logic**
  - why reserved: Core execution capability.
  - risk if handled by weak model: Unauthorized task execution.
  - approval required: Explicit routing logic token.
  - current status: Reserved
  - runtime effect if authorized later: Execution capability enabled.

- **queue/routing transition logic**
  - why reserved: Complex state synchronization.
  - risk if handled by weak model: Deadlock or race conditions.
  - approval required: Explicit orchestration token.
  - current status: Reserved
  - runtime effect if authorized later: Live routing enabled.

- **external tool integration**
  - why reserved: Network/API attack surface.
  - risk if handled by weak model: Credential exfiltration.
  - approval required: External tool token.
  - current status: Reserved
  - runtime effect if authorized later: Network/API enablement.

- **credential/secret denial logic**
  - why reserved: Sensitive security layer.
  - risk if handled by weak model: Security compromise.
  - approval required: Security token.
  - current status: Reserved
  - runtime effect if authorized later: Access control change.

- **API/network denial logic**
  - why reserved: Network safety boundary.
  - risk if handled by weak model: Network boundary leak.
  - approval required: Security token.
  - current status: Reserved
  - runtime effect if authorized later: Security logic change.

- **production execution candidates**
  - why reserved: Highest blast radius.
  - risk if handled by weak model: Production outage.
  - approval required: Production gate token.
  - current status: Reserved
  - runtime effect if authorized later: Production state change.

- **architecture refactors**
  - why reserved: Cross-layer system impact.
  - risk if handled by weak model: Broad system corruption.
  - approval required: Operator-wide architecture token.
  - current status: Reserved
  - runtime effect if authorized later: Systemwide change.

## High-Model Does Not Mean Authorized
- top-tier model availability does not authorize runtime work
- top-tier model availability does not authorize v4.8
- top-tier model availability does not authorize API/network
- top-tier model availability does not authorize deployment
- top-tier model availability does not authorize production
- explicit operator instruction is still required

## Station Chief Resume Conditions

Station Chief runtime resumes only when the operator explicitly assigns a Station Chief runtime task.

Required explicit resume markers may include:
- direct assignment to build v4.8
- direct assignment to fix runtime
- direct assignment to modify validators
- direct assignment to resume Station Chief ladder

Do not treat general discussion as resume authorization.
