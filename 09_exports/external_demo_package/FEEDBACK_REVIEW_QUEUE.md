# Feedback Review Queue Guide — Operator Instructions

Once feedback packets are returned by external reviewers, they enter the manual review queue.

## Triage Process
1. **Import:** Paste the returned text packet into the local operator review surface.
2. **Review:** Categorize the signal (Product, Technical, Safety, Pitch).
3. **Follow-up:** Identify if a direct question was asked that needs a response.
4. **Task Conversion:** Convert high-signal feedback into future MVP candidate tasks.

## Manual Queue States
- **Pending Review:** New packet received, not yet categorized.
- **Reviewed:** Signal captured, no immediate action needed.
- **Needs Followup:** Waiting for clarification from the reviewer.
- **Converted:** Feedback is now an official project task/milestone.
- **Archived:** Duplicate or low-signal feedback.

## Automation Policy
- **NO AUTO-EXECUTION:** Feedback cannot automatically trigger code changes or deployments.
- **NO AUTO-APPROVE:** Feedback does not automatically "pass" a phase; operator review is mandatory.
